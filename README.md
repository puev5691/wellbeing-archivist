# Архивариус проекта Благополучие

Локальный MVP-инструмент для индексирования, поиска, учёта и постепенного наведения порядка в файловом хранилище проекта Благополучие.

Проект вырос из практической проблемы:
когда чаты, документы, черновики, журналы, handoff-файлы и проектные материалы начинают накапливаться, простого просмотра каталогов становится недостаточно.

Архивариус предназначен для того, чтобы помогать:

- индексировать файлы;
- искать по имени и по тексту;
- распознавать пакеты чатов;
- работать с метаданными пакета;
- читать карты раскладки файлов;
- видеть, где пакет ещё не оформлен до конца;
- подсказывать, чем закрывать дыры в карте раскладки.

## Текущее состояние

Архивариус уже не является только проектной идеей.

На текущем этапе он работает как локальный CLI-инструмент с SQLite-базой и практической поддержкой реального рабочего пакета чата.

Реализованные возможности:

- инициализация SQLite-базы
- полная индексация и переиндексация
- поиск по имени файла
- поиск по текстовому содержимому
- распознавание пакета чата
- чтение `.wb-copy-map.tsv`
- показ разобранной карты раскладки
- генерация черновика карты раскладки
- мягкая классификация файлов пакета
- проверка покрытия карты раскладки
- рекомендации по пополнению `.wb-copy-map.tsv`

## Зачем нужен Архивариус

В проекте используется подход:
один чат = один рабочий пакет.

Пакет чата может содержать:

- входные файлы контекста
- handoff-файлы
- сгенерированные документы
- журналы
- shell-скрипты
- служебные файлы пакета
- карту дальнейшей раскладки по каноническим каталогам

Архивариус нужен для того, чтобы такие пакеты переставали быть просто хаотичными папками и становились:
- обозримыми;
- проверяемыми;
- пригодными к поиску;
- пригодными к постепенной дисциплине хранения.

## Базовые сущности текущего MVP

## Типовой рабочий цикл

Минимальный практический сценарий работы выглядит так:

1. инициализировать базу
2. выполнить индексацию или переиндексацию пакета
3. посмотреть карточку пакета
4. посмотреть карту раскладки
5. классифицировать файлы пакета
6. проверить покрытие карты раскладки
7. сгенерировать черновик карты или рекомендации
8. вручную подтвердить и обновить `.wb-copy-map.tsv`

Примеры команд:

    python archivist.py --config config/real_chat_package_config.json init-db
    python archivist.py --config config/real_chat_package_config.json reindex
    python archivist.py --config config/real_chat_package_config.json show-chat-package 1
    python archivist.py --config config/real_chat_package_config.json show-copy-map 1
    python archivist.py --config config/real_chat_package_config.json classify-package-files 1
    python archivist.py --config config/real_chat_package_config.json check-copy-map-coverage 1
    python archivist.py --config config/real_chat_package_config.json recommend-copy-map-additions 1

## Что умеет работа с картой раскладки

На текущем этапе `.wb-copy-map.tsv` рассматривается как карта маршрутизации файлов из рабочего пакета чата в канонические каталоги проекта.

Архивариус уже умеет:

- читать существующую карту раскладки
- показывать её строки
- генерировать черновик карты по составу пакета
- проверять, какие файлы уже покрыты картой
- рекомендовать новые строки для непокрытых файлов

Архивариус пока не умеет:

- автоматически утверждать окончательную судьбу файла
- автоматически копировать файлы в канонические места
- глубоко различать смысловой статус каждого документа
- надёжно отличать итоговый артефакт от промежуточного черновика во всех случаях

Это сделано намеренно.
На текущем этапе Архивариус должен помогать дисциплинированным человеческим решениям, а не заменять их самоуверенным бредом.

## Зависимости

Текущий MVP использует только стандартную библиотеку Python.

Для реализованной CLI-функциональности внешние runtime-зависимости не требуются.

## Направление дальнейшего развития

Ближайшие направления развития:

- более безопасное полуавтоматическое обновление `.wb-copy-map.tsv`
- лучшее различение итоговых файлов и промежуточных черновиков
- более выразительные карточки пакетов
- усиление извлечения метаданных из шапок markdown-файлов
- развитие маршрутизации между пакетами и каноническими контурами
- в будущем GUI или более высокий операторский интерфейс
