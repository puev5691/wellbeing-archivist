# wellbeing-archivist

Локальный MVP-архивариус для файлового хранилища проекта Благополучие.

На текущем этапе проект умеет:
- создавать SQLite-базу;
- индексировать учебное дерево каталогов;
- искать файлы по имени;
- искать файлы по тексту;
- различать базовые контуры проекта;
- находить простые пакеты чатов внутри `00_INBOX`.

## Текущий статус

Статус: ранний MVP-каркас

Это ещё не полноценная система ревизии и не GUI-приложение.
Сейчас задача проще:
получить рабочий локальный индексатор и безопасно обкатать его на учебных данных.

## Структура запуска

Основная точка входа:

`archivist.py`

Основной конфиг для учебного прогона:

`config/default_config.json`

## Что нужно перед первым запуском

Нужен Python 3.

Если запуск идёт в Termux, проверь наличие Python:

    python --version

## Первый безопасный прогон

### 1. Перейти в каталог репозитория

    cd /storage/emulated/0/Documents/repos/wellbeing-archivist

### 2. Сделать скрипты исполняемыми

    chmod +x scripts/make_sample_data.sh
    chmod +x scripts/run_smoke_test.sh

### 3. Запустить тренировочный прогон

    bash scripts/run_smoke_test.sh

## Что делает smoke test

Скрипт:

1. создаёт учебное дерево каталогов в `data/sample/Blagopoluchie`;
2. создаёт тестовый пакет чата;
3. создаёт несколько markdown-файлов в разных контурах;
4. инициализирует SQLite-базу;
5. запускает индексацию;
6. показывает статистику;
7. выполняет поиск по имени;
8. выполняет поиск по тексту.

## Ожидаемый результат

После успешного прогона должно быть видно:

- база создана;
- индексатор обработал файлы;
- найден тестовый файл по имени `test`;
- найден файл по слову `архивариус`;
- в статистике появилось количество файлов и пакетов чатов.

## Ручные команды

### Инициализация базы

    python archivist.py --config config/default_config.json init-db

### Индексация

    python archivist.py --config config/default_config.json index

### Статистика

    python archivist.py --config config/default_config.json stats

### Поиск по имени

    python archivist.py --config config/default_config.json search-name "test"

### Поиск по тексту

    python archivist.py --config config/default_config.json search-text "архивариус"

## Важное ограничение текущего этапа

Пока не надо сразу натравливать Архивариуса на всё боевое хранилище.
Сначала нужно убедиться, что он:

- не падает на базовом прогоне;
- корректно создаёт базу;
- нормально читает тестовые markdown-файлы;
- видит пакет чата;
- возвращает ожидаемые результаты поиска.

Только после этого имеет смысл делать отдельный конфиг под реальные каталоги проекта.

## Следующий шаг после успешного smoke test

После успешного учебного прогона логично делать:

1. конфиг для реального хранилища;
2. более аккуратный вывод результатов;
3. обработку `.wb-copy-map.tsv`;
4. тесты;
5. улучшение поиска;
6. потом уже всё остальное, что так любит распухать раньше времени.
